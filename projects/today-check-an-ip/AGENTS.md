# Codex 工作规则：今天验一个 IP

本目录是《今天验一个 IP》内容项目。

## 默认语言

所有面向用户的内容默认使用简体中文。

## 写作风格

- 从真实生活和业务场景进入，不从技术定义进入。
- 用普通人能听懂的话解释网络问题。
- 段落完整，避免一句一行和过度换行。
- 少用术语；必须出现术语时，先解释它解决什么问题。
- 不装腔，不神化技术，不制造恐慌。
- 结论直接，允许明确说“不建议”“信息不足”“不能仅凭这一项判断”。
- 不使用“防封”“养号神器”“百分百纯净”“平台认可”等无法验证的宣传话术。

## 分析 IP 时的固定思路

先回答“它是谁”，再回答“它表现怎么样”。

优先检查：

1. ASN
2. ASN 所有者 / 运营商
3. 网络类型：ISP、IDC、教育、企业等
4. 注册地区与多个常见定位结果的一致性
5. 是否存在代理、VPN、托管网络等公开识别信号
6. 公开风险、垃圾邮件、扫描或滥用信号
7. 检测网站之间是否存在明显冲突
8. 数据的新旧和可解释性

不得把任意单一检测网站的“纯净度”“风险分”直接当作事实。

## 结论等级

每一期最终只给以下四类判断之一：

- 可进入测试
- 建议继续观察
- 谨慎使用
- 我不会交付给客户

判断必须写清依据。

## 内容结构

每篇内容尽量保持：

场景 → 用户疑问 → 拆检测结果 → 看 ASN → 看运营商和类型 → 对照其他信息 → 我的判断 → 风险边界 → 一个自然互动问题

## 隐私规则

公开内容中的 IP 默认隐藏最后一段，例如：`121.52.64.xxx`。

不得公开用户账号、店铺、设备标识、代理用户名密码、节点链接、API Key 或其他凭据。

## 项目目标

当前目标是连续完成 10 期真实内容，观察：

- 哪类标题带来点击
- 哪类问题带来收藏
- 哪类 IP 案例带来评论
- 哪类内容带来真正的网络环境咨询

不要过早把项目改造成复杂 SaaS、检测平台或自动化系统。先验证内容和需求。
